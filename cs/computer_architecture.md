데이터 버스: 명령어와 데이터를 주고받는 통로
제어버스 :  제어 신호를 주고받는 통로

CPU가 주소 버스로 접근할 위치를 보내고, 제어 버스로 읽기/쓰기 동작을 지정한 다음, 데이터 버스로 실제 명령어나 데이터를 주고받는다. 이 세 버스가 함께 동작해서 하나의 메모리 접근이 이루어진다.---> 이게 하나의 동작 방식이다.

메모리 : (현재 실행되는) <-- 이게 핵심이다. 프로그램의 명령어와 데이터를 저장하는 부품

메모리는 실행되는 프로그램의 명령어와 데이터를 저장함

이떄 메모리는 각자의 주소가 있음 (내가 사용하고자 하는 명령어 데이터가 있는 곳)

cpu : 메모리에 저장된 명령어를 읽어 들이고 해석하고 실행하는 부품이다.

제어신호란? :  컴퓨터의 부품들을 관리하고 작동시키기 위한 전기 신호 이 신호에는 2가지 :

cpu가 메모리에 어떤 값을 저장하고 싶을 땐 메모리를 향해 : 메모리 읽기,

 반대로 어떤 값을 저장하고 싶을 떈 : 메모리 쓰기

 pc는 명령어주소를 가짐---> cpu가 그 주소를 mar(접근 주소 저장)에 넣고 주소 버스를 통해 메모리 전달 ---> 메모리에서 명령어 가져옴 (이때 Memory data register를 거침 )
 ---> 명령어가 ir에 들어감 ---> 제어장치가 ir을 해독함  ---> 그래서 연산에 필요한 데이터를 다시 메모리에서 가져옴 (register)에 다시 저장 --> 메모리에서 해당 데이터를 가져옴 ---> alu가 계산함 ---> 그 결과를 저장함 (레지스터)---> 그 이후 다시 명령어를 메모리에서 가지고옴 그 가지고 온 명령어를 다시 제어장치에서 해독---> 레지스터에서 꺼내서 그 데이터를 빈 메모리에 저장 

 보조 기억 장치: ram은 저장된 메모리를 다 잃어버리는 문제가 발생함 (휘발성) ---> 그래서 전원이 꺼져도 보관될 프로그램을 저장하는 부품
